Posted on 2019-10-10 In 算法设计 Views:

求一个字符串中的最长重复字串

题目

Given a string S, consider all duplicated substrings: (contiguous) substrings of S that occur 2 or more times. (The occurrences may overlap.)

Return any duplicated substring that has the longest possible length. (If S does not have a duplicated substring, the answer is "".)

Note:

  1. 2 <= S.length <= 10^5
  2. S consists of lowercase English letters.

Example

思路

题目就是找重复的最长局部子串，先直观提出一个重要的特点：

  • 最长重复子串包含其他重复字串

博主我第一个想到的是kmp么，先写写看咯，反正再优化嘛。。。一开始想的是利用DP，开辟数组存每一步的结果，如果短的字符串能找到，那么长的就直接在原来的基础上找就可以了。掐指一算，嗯。。。$O(s^2*t)$的垃圾算法。

改进一下，其实不用再次匹配（省掉t的时间，就是$O(s^2)$），因为求next就是求了最大字串的长度，next数组最大的数字对应的位置就是所求了。这下也省了一些空间，感觉挺好的嘛，只是next是求前缀和后缀相等的长度，要求所有重复子串，就要循环erase掉首字母。具体看解答。总之时间上没有过。

过不了其实可以预见，因为没有利用上面说的那个特点。

于是学习了两个神奇的工具：Rabin–Karp 和 Suffix-Array。

前者Rabin–Karp的方法思路是把字符串转化成数字，每一个字串放在hash表中，利用一个窗口动态计算每一个位置的字符的数字，如果有相等的的 value ，则进一步判断（因为hash不保证绝对相等）。同时还可以利用重复子串的局部也重复的属性，这个窗口还可以用二分法来调整，如果一个窗口能找到两个相等字符串，则可以调大；否则调小。单次匹配并没有 KMP 优雅，然而相比之下做多模匹配的优势简直不能再大。看下面代码。

后者 Suffix-Array 原理很简单，代码很抽象，脑阔很疼。。。原理是利用倍增（k*=2）循环，每次对字符串中的一个部分进行基数排序，下一次循环将上一次的相邻结果合并（也就是相邻部分的字符串合并），合并之后看作一个字符串再次进行排序，如此倍增循环，得到后缀数组。由于最长共同前缀的高度（在树中）可以由 suffix-array 很简单计算出来，而两者就结合很容易得到答案。看下面代码（参考JAVA大佬写的C++，是我觉得所有 suffix-array 代码里思路最清晰，变量最简洁的了，比经典的一大坨循环版本舒服很多= =、）。

寻找两个等长有序数组的中位数

题目

There are two sorted arrays A and B of size n . Find the $n^{th}$ number of the two sorted arrays. The overall run time complexity should be O(log (n)).

这里求中位数和第n个数其实等价。

Example

思路

考虑中位数的特点，有一个长$2n$的有序数组，$n^{th}$的数前面有$n-1$个比它小的数，后面有$n$个比它大的数。那么在两个有序的长度为$n$的数组$a,b$中，找一个$n^{th}$的数，也就是找一个数，排在它之前的数加上另一个数组比它小的数的和为$n - 1$，而且排在之后的数与另一个数组比它大的数的和为$n$。这两个界锁定的时候就是程序的出口了。现在思路就很清楚了，先初始化两个最宽松的界，然后缩小这两个界，就找到了这个中位数。这里分治的思想主要体现在分别缩小上下界。

寻找两个不等长有序数组的中位数

题目

There are two sorted arrays nums1 and nums2 of size m and n respectively. Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

注意这里中位数可能不存在于数组里。

Example

思路

这里和上一题的区别是这里是不等长数组。沿着上一题的思路，我们还是用缩小上下边界的方案。但是需要注意的是，由于上一题是等长数组，想等长度给我们同时操作两个数组的上下界提供了便利，而不等长数组操作上下界的时候需要注意越界的问题，通过修改短数组的界来驱动改变长数组的界可以避免这个问题。同时还有一个非常恶心的地方是，它的中位数因奇偶的情况，可能在数组中也可能是两个数的平均，导致最后需要判断更多的情况。

循环缩小边界的方案需要非常细心，博主整了几个小时，也没能让最后的两组上下界收敛到一个可以应对所有奇偶情况的状态，因为随便写几个条件就变得很非常不优雅，而且还是不能pass，博主就放弃了。结果看了别人的循环，emmmmmmm，确实不优雅哈哈哈。。。。

虽然能抓得到老鼠的猫才是好猫，但是我就是喜欢好看的猫啊= =、所以改造一下上一题递归的算法，使用找$k^{th}$的方案，找中位数即变成了求$\frac{1}{2}\times(ab[\frac{total}{2}]+ab[\frac{total-1}{2}])$的数，$ab$表示两个数组的$merge$。这里需要注意的是，$vec::erase$操作理论复杂度是$O(n)$，可以考虑用标记位取代删除操作。博主经过测试运行时间，这里只有删头操作，是经过了优化，没有对整体时间造成影响，所以就这么写了。

寻找峰形数组中的某个数

题目

You may recall that an array A is a mountain array if and only if:

  • A.length >= 3

  • There exists some i with 0 < i < A.length - 1

    • A[0] < A[1] < ... A[i-1] < A[i]
    • A[i] > A[i+1] > ... > A[A.length - 1]

    Given a mountain array mountainArr, return the minimum index such that mountainArr.get(index) == target. If such an index doesn’t exist, return -1.

You can’t access the mountain array directly. You may only access the array using a MountainArray interface:

  • MountainArray.get(k) returns the element of the array at index k (0-indexed).

    • MountainArray.length() returns the length of the array.

    Submissions making more than 100 calls to MountainArray.get will be judged Wrong Answer. Also, any solutions that attempt to circumvent the judge will result in disqualification.

也就是有一个元素互不相同的峰形数组，要求某个数是否在其中，返回其最小的小标（存在）或$-1$（不存在）

Example

思路

它简单可以当成二分查找查找峰值位置$+$两个有序数组的二分查找两个问题，但这里有一点需要注意，$ma.get()$的使用次数有限制。所以可能没这么简单，需要优化一下过程。

事实上我们需要找的数不一定需要准确的山峰值，如果我们找到左坡和右坡比它大的数，就是一个合适的“假峰”了。这里我们还是先找山峰，过程中调整左右“假峰”。最后分别对左，右二分查找即可。
